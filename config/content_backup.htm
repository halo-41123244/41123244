練習

Tutorial1

教學目的

  • 本章的目的是解說 Webots 的基本概念，這些概念在自行創建模擬場景 (.wbt) 時必須用到。

  • 讓你熟悉機器人及其環境的模型建構流程，以及機器人控制器 (controllers) 的程式設計。

  • 本章適合 Webots 的初學者，但仍然需要有程式設計背景（範例使用 Python）

  • 教學流程被設計為「導引課程」形式，每個子課程都有明確教育目標、互動操作步驟，以及結論小節。

重點內容整理

  • 新世界 (World) 的建立：包括地板、牆壁、障礙物 (箱子) 的添加。

  • 機器人 (以 e-puck 為範例) 的加載、位置與旋轉設定。

  • 控制器程式 (使用 Python) 的創建：取得機器人與馬達裝置、設置時間步長、控制輪子移動。

  • 模擬世界的保存與運行：保存 .wbt 檔案後執行模擬並觀察機器人動作。

  • 本章為後續教程奠基：熟悉基本操作後，可進一步探索 Webots 的進階功能。

教學流程摘要

  1. 啟動 Webots 軟體。

  2. 創建新世界：File → New World，並命名檔案 (如 my_first_simulation.wbt)。

  3. 添加競技場 (arena)：在場景樹 (Scene Tree) 裡新增 Solid → Shape，設定 Box 的尺寸作為地板，並指定材質與顏色。

  4. 添加牆壁：於競技場周圍建立四面牆，設定位置與尺寸使其包圍地板。

  5. 添加箱子 (障礙物)：在競技場中放置多個箱子，設定其尺寸、位置，並可複製移動箱子到不同位置。

  6. 添加 e-puck 機器人：在場景樹中新增 Robot，從列表中選擇 e-puck，並設定其位置與旋轉。

  7. 創建控制器程式：

    • 在場景樹下的 e-puck 上新增 Controller，選擇使用 Python。

    • 在程式中導入 Robot、Motor 類別，設定 TIME_STEP，取得左右輪電機裝置，設置輪子目標位置，使機器人移動。

  8. 保存世界，點擊 Play 運行模擬，觀察機器人的動作。

Tutorial2

教學目的

  • 本章目的是在 Webots 中建立一個更具互動性的模擬環境：學會加入物件（如球）、設定物理特性，並「修改環境」以利後續機器人與環境互動。

  • 讓你熟悉節點（nodes）概念：知道「節點」在 Webots 世界 (.wbt) 中代表什麼、如何新增、隸屬關係如何、以及物理節點設定。 

  • 學會使用 “DEF-USE” 機制來重用節點定義，從而減少重複、提升場景效率。

重點內容整理

  • 刪除預設的地板部件 (RectangleArena)，手動新增一個新的地板 (Floor) 並調整其尺寸為 1 m x 1 m。 

  • 介紹關鍵節點：

    • Solid 節點：代表剛體（rigid body），Webots 的物理引擎對剛體模擬較為適合。 

    • Shape、Sphere、Box 節點：分別用於定義視覺幾何與物理邊界，以及各種類型的幾何體。 

    • Physics 子節點：用於定義物體是否為「動態」（會受重力／碰撞影響）或「靜態」。 

  • 在場景中新增一個球 (Sphere) 作為剛體：

    • 建立 Solid → Shape → PBRAppearance → Sphere 節點。設定其 radius（如 0.05 m）、subdivision 等參數。 

    • 使用 boundingObject 欄位用 Sphere 定義碰撞邊界。 將球放置於機器人前方，例如在位置 {0.2, 0, 0.2}。 

  • 引入 DEF-USE 機制：

    • 在場景中將一個幾何（如 Sphere）用 DEF 標記一名稱（如 BALL_GEOMETRY），然後在其他位置透過 USE 引用這定義，以避免重複。

    • 示範如何將先前兩個 Sphere 節點整合成一個定義 + 多處重用。 

  • 新增四面牆 (Box 幾何) 圍繞環境：這些牆設定為靜態物件（無物理效果），用以界定模擬區域。 

  • 性能提示：剛體模擬計算成本較高，建議減少邊界物件數量、限制約束、提高 WorldInfo.basicTimeStep 欄位以提升模擬速度。 

教學流程摘要

  1. 開啟你在第 1 章創建的 my_first_simulation.wbt 世界檔，確保模擬暫停、虛擬時間為 0。接著使用 File → Save World As… 將其另存為 obstacles.wbt。 

  2. 刪除預設的 RectangleArena 節點：在場景樹或 3D 視圖選中該節點，然後刪除。

  3. 新增一個 Floor (Solid) 節點作為地板：在場景樹中加入 TexturedBackgroundLight → Floor (Solid) 節點。設定 size 欄位為 {1, 1} 以將地板調整為 1 m × 1 m。 

  4. 在地板之前、機器人前方新增一個球體：

    • 新增 Solid 節點 → 於其 children 欄位新增 Shape 節點，Shape 的 geometry 欄位新增 Sphere 節點；

    • 為 Shape 的 appearance 欄位新增 PBRAppearance 節點，並設定 metalness = 0、roughness = 1；

    • 在 Solid 的 boundingObject 欄位新增另一 Sphere 節點作為碰撞邊界；

    • 在 Solid 的 physics 欄位新增 Physics 節點。

    • 將這個球體翻譯 (translation) 到位置如 {0.2, 0, 0.2}。 

  5. 調整球體細節：對兩個 Sphere 節點分別設置 radius = 0.05 m、subdivision = 2。

  6. 使用 DEF-USE 機制：

    • 為第一個 Sphere 的欄位輸入 DEF 字串（如 BALL_GEOMETRY）；

    • 刪除 boundingObject 裡的第二個 Sphere；

    • 在 boundingObject 中選擇 Add → USE / BALL_GEOMETRY。此後修改第一個 Sphere 的 radius 欄位會同步影響 boundingObject。 

    • 再新增第二個球，使用相同參數但透過 Shape 節點來實現 DEF-USE 重用。 

  7. 新增四面靜態牆壁來圍繞環境：使用單一 Box 幾何定義，並設定為靜態（無 physics 欄位或 physics = NULL）。 

  8. 保存模擬並可比較參考解答檔案：世界檔案可使用文字編輯器打開比對。 

  9. 結論：經過本章學習，能夠基於剛體 (Solid) 創建簡單環境、從場景樹視圖新增節點並修改欄位，熟悉了如 Solid、Shape、Sphere、Box 節點與 DEF-USE 機制。

Tutorial3

教學目的

  • 本章的目的是學習 Webots 中與 外觀（Appearance） 及 照明（Lighting） 相關的節點，讓模擬環境更加真實。

  • 學會設定物體的顏色、質感與反光特性，並瞭解不同光源對場景呈現的影響。理解良好的視覺外觀不僅影響美觀，也影響模擬中相機或感測器的運作（例如顏色辨識與循線功能）。

  • 教學結構延續前章，重點在環境視覺層面的提升與節點屬性設定。

重點內容整理

  • 從上章的 obstacles.wbt 世界檔另存為新檔 appearance.wbt。

  • 認識 Webots 中三種光源節點：

    • DirectionalLight：平行光（模擬太陽光），光線方向固定。

    • PointLight：點光源（如燈泡），光線向四面八方擴散。

    • SpotLight：聚光燈（如手電筒），有光錐角與方向。

  • 使用 TexturedBackgroundLight 節點，自動結合背景與光源，產生自然的全域光照效果。

  • 學習 Appearance 與 PBRAppearance 節點的差別：

    • Appearance 為傳統光照模型 。

    • PBRAppearance 採用物理式渲染模型 (Physically-Based Rendering, PBR)，提供更真實的金屬質感與粗糙表面效果。

  • 可透過調整 baseColor, metalness, roughness 屬性改變物體表面反射與亮度特性。

教學流程摘要

  1. 開啟上章完成的 obstacles.wbt，並使用 File → Save World As… 將其另存為 appearance.wbt。

  2. 確認場景樹中包含 TexturedBackgroundLight 節點。此節點自帶方向光 (DirectionalLight) 與背景貼圖，能自動產生柔和的照明效果。

  3. 若想手動控制照明，可新增 Light 節點：

    • 在根節點 (Root) 下新增 DirectionalLight 或 PointLight。

    • 可調整 color、intensity、direction 等屬性。

  4. 選取牆壁的 Shape 節點，新增外觀設定：

    • 在 appearance 欄位選擇新增 PBRAppearance。

    • 修改 baseColor 為淡藍色或其他色彩。

  5. 調整反光屬性：

    • 設定 metalness（金屬感）為 0～1 間的數值。

    • 設定 roughness（粗糙度）為 0～1 間的數值。

    • 觀察場景中反光與陰影變化。

  6. 可以為地板或牆壁加入不同顏色與反光設定，模擬出不同材質效果。

  7. 保存世界檔，點擊 Play 以檢視最終的照明與外觀效果。

結論

  • 學會在 Webots 中使用光源與外觀節點，創建更具真實感的模擬環境。

  • 透過 PBRAppearance 調整顏色、金屬度與粗糙度，可以精確控制物體的視覺效果。

  • 理解光源數量與渲染品質之間的平衡，能協助建立兼顧美觀與效能的模擬場景。

Tutorial4

教學目的

  • 本章的目的是介紹 Webots 控制器（controller）編程的基礎概念與實作流程。

  • 學會如何讓機器人 感知環境（透過感測器） 並 執行動作（透過馬達），以實現簡單的自動行為控制。

  • 理解場景樹（Scene Tree）節點與控制器 API 之間的關聯，學會初始化與清理控制器、初始化設備、讀取感測值、控制致動器，並建立簡單的 反饋迴路（feedback loop）。

  • 教學著重於 Webots 功能的正確使用，不探討進階的機器人演算法。

重點內容整理

  • 新世界檔案與控制器建立：

    • 將前章的世界另存為 collision_avoidance.wbt。

    • 透過 File → New → New Robot Controller… 建立新控制器（例如 epuck_avoid_collision）。

    • 在 e-puck 節點中修改 controller 欄位以連結新控制器。

  • e-puck 模型介紹：

    • 配備 8 個紅外距離感測器（ps0 到 ps7），數值範圍 0–4096。

    • 感測器越接近障礙物，輸出值越大。

    • 由 DistanceSensor 節點建模，可透過 webots/distance_sensor.h API 存取。

  • 控制器 API 概念：

    • 提供介面讓使用者能夠存取模擬中的感測器與執行器。

    • 每個設備透過名稱（WbDeviceTag）呼叫 wb_robot_get_device() 取得。

    • 感測器需透過 .enable(time_step) 啟用。

  • 程式語言可用 C、C++、Java、Python，範例使用 Python。

  • 控制邏輯：

    • 機器人向前移動，若前方感測器偵測到障礙物，則轉向無障礙物的一側。

    • 以 UML 狀態機示意控制流程（感測 → 判斷 → 動作）。

Tutorial5

教學目的

  • 本章的目的是透過在 Webots 中創建一個具有複雜形狀（由多個子幾何構成）的固體，來深入探索物理模擬的設定與控制。 

  • 目標是能夠設計並建立一個由兩個球體與一根圓柱體構成的「啞鈴」模型，並了解其物理屬性如何影響模擬行為。 

  • 精通 Solid 節點中多個 Shape 子節點的使用方法、Pose 節點的定位與變換、以及將圖形與物理形狀合併。 

重點內容整理

  • 從上一章的世界檔案另存為 compound_solid.wbt。 

  • 創建一個複合固體（Compound Solid）：

    • 使用一個 Solid 節點，內部包含多個 Shape 節點來定義圖形與物理外形。

    • 每個 Shape 節點可用 Pose 子節點調整其 相對位置與方向。 

  • 啞鈴模型實作：

    • 圓柱體（Cylinder）作為啞鈴桿：長度為 0.1 m，半徑為 0.01 m。 

    • 兩個球體（Sphere）作為啞鈴重量端：球的半徑為 0.03 m，subdivision 設為 2。 

    • 將兩個球體透過其 Pose 節點移動至桿的兩端。 

  • 補充說明：

    • Solid 節點可包含不少於一個 Shape，且這些 Shape 可同時定義視覺與物理邊界。 

    • Group 節點亦可用來將多個節點組合起來，以構建更複雜的結構。 

教學流程摘要

  1. 開啟你在 Tutorial 4 所創建的世界，然後使用 File → Save World As… 將其另存為 compound_solid.wbt。

  2. 在場景中新增一個 Solid 節點作為啞鈴的初始父節點。

  3. 在該 Solid 裡新增一個 Cylinder Shape 節點：設定長度為 0.1 m、半徑為 0.01 m。

  4. 在同一 Solid 中新增兩個 Sphere Shape 節點：設定球半徑為 0.03 m、subdivision = 2。

  5. 對每個球體 Shape，再各自使用一個 Pose 節點來設定其 translation（位置）使其位於桿的兩端。

  6. 儲存並執行模擬，觀察固體是否如預期地構成啞鈴結構，同時確認其物理互動是否合理。

  7. （可選）打開世界檔案為文字檢視，了解 Solid 與其子節點的階層與欄位定義。

結論

  • 本章你學會如何在 Webots 裡構建「多幾何構成的固體」，也了解了 Solid、Shape、Pose、Group 等節點在場景中的組合方式與作用。

  • 你已掌握構造複合固體所需的關鍵參數（如長度、半徑、subdivision、translation）與節點關聯。

  • 下一步，你可以將這些結構應用於設計更複雜的模擬場景，並開始加入動力學與碰撞互動的設定。

Tutorial6

教學目的

  • 本章的目的是從頭開始創建一輛 四輪機器人 (4‑Wheeled Robot)，包括車體、四個車輪、兩個距離感測器。

  • 讓你理解在 Webots 中從模型設計到控制器程式的完整流程：包括節點結構、車輪關節機構、感測器配置與程式控制。

  • 幫助你掌握建構機器人模型時 Solid／Joint／Device 節點的用途與組織方式，並在控制程式中驅動車輪、讀取感測器值並做出反應。

重點內容整理

  • 新模擬場景：將上一章結果保存為 4_wheeled_robot.wbt，並刪除舊的 e‑puck／球體／啞鈴與其接觸屬性，只保留地面、牆壁、燈光。

  • 機器人模型設計要點：

    • 整車由 5 個 Solid 節點構成：車體 + 四個車輪。

    • 車輪透過 HingeJoint 節點與車體連接。Solid、Joint、Device 的階層與組織須遵循 Webots 資料結構。

  • 車體設定：使用 Box 幾何，尺寸 (0.2, 0.1, 0.05) m，顏色紅色。車體節點需加入 Physics。

  • 車輪設定：每輪為 Cylinder 幾何，半徑 0.04 m、厚度 0.02 m。每輪透過 HingeJointParameters 指定 anchor、axis，其中軸為 (0,1,0)。車輪節點顏色綠色。

  • 感測器：新增兩個 DistanceSensor 節點為車體子節點，感測方向沿 +x 軸，並旋轉 0.3 rad 與車前方產生角度。感測器圖形用邊長 0.01 m 的 Cube，顏色藍色。

  • 控制器程式設計：導入 motor 模組，取得四輪 Motor 裝置，設定位置為無限旋轉 (setPosition(float('inf')))，然後設置速度。程式中讀取距離感測器值，若感測值低於某閾值表示前方有障礙，則啟動避障邏輯。

  • 測試與解決方案：可開啟 robot‑window 實時觀察感測器與馬達值。程式名稱例如 four_wheeled_collision_avoidance，並在 Robot 節點中設定其 controller 欄位。

教學流程摘要

  1. 開啟上一個世界檔案，另存為 4_wheeled_robot.wbt。

  2. 刪除不再使用的舊模型元素（e‑puck、球、啞鈴、接觸屬性），保留基本環境（地面、牆壁、燈光）。

  3. 在場景樹末端添加一個 Robot 節點，並作為整車根節點。

  4. 在 Robot 節點下，添加四個 HingeJoint 節點，每個 HingeJoint 的 endPoint 為一個 Solid → Shape (Cylinder) 用於車輪。設定 Cylinder 的尺寸、顏色、位置與旋轉。為 HingeJoint 設定 HingeJointParameters 的 anchor 與 axis。

  5. 車輪 Motor 裝置命名為 wheel1 ~ wheel4，取得並初始化。

  6. 添加兩個 DistanceSensor 裝置，設置其名稱、位置、旋轉、圖形形狀（cube）、顏色。啟用它們以讀取距離。

  7. 寫控制器程式：設定 TIME_STEP，如 64 ms。初始化 robot、感測器、馬達。主迴圈中讀取感測器值、判斷障礙、調整左右輪速度。

  8. 保存檔案、設定 Robot 節點的 controller 欄位為控制器程式名稱，運行模擬並觀察機器人的避障行為。

  9. 比對官方解答世界檔（位於 solution 資料夾）以確認模型設計與控制器程式正確。

結論

  • 完成本章後，你能夠從零設計、建構並控制一輛四輪機器人。

  • 你已熟悉 Solid／Joint／Device 節點在 Webots 中的用途、如何構建機器人模型、以及如何編寫控制器驅動與感測。

  • 這為後續加入更多感測器、控制策略或更複雜機器人奠定了基礎。

Tutorial7

目標

將上一個教學中建立的四輪機器人從世界檔 (.wbt) 移到 PROTO 檔案，方便在不同世界或多次使用同一機器人，而不需要複製整個定義。

1. 建立 PROTO 檔案

  1. 開啟上一個世界檔 4_wheels_robot.wbt。

  2. 在 protos 資料夾中建立新檔 FourWheelsRobot.proto。

  3. PROTO 檔案的基本結構：

    • protoName：PROTO 名稱，例如 FourWheelsRobot

    • protoFields：可修改的欄位（fields），目前可以先留空

    1. 將原本世界檔中的 Robot { … } 節點複製到 protoBody。

2. 在世界中使用 PROTO

  • 儲存 PROTO 檔後，重新開啟 Webots。

  • 可以在 PROTO nodes (Current Project) 中找到 FourWheelsRobot 並新增到世界中。

  • 可以新增多個相同的機器人節點。

3. 新增可修改欄位（Fields）

  1. 編輯 PROTO 檔案，增加可調整的欄位：

    field SFVec3f translation 0 0 0
    field SFRotation rotation 0 0 1 0
    field SFFloat bodyMass 1

4. 小結

你現在學會了：

  1. 將 Webots 節點定義轉換成 PROTO。

  2. 在 PROTO 中開放欄位（fields）供外部修改。

  3. 連結 PROTO 欄位與內部節點屬性 (IS 關鍵字)。

這樣可以讓機器人更具模組化與可重用性。
