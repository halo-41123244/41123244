Tutorial4

教學目的

  • 本章的目的是介紹 Webots 控制器（controller）編程的基礎概念與實作流程。

  • 學會如何讓機器人 感知環境（透過感測器） 並 執行動作（透過馬達），以實現簡單的自動行為控制。

  • 理解場景樹（Scene Tree）節點與控制器 API 之間的關聯，學會初始化與清理控制器、初始化設備、讀取感測值、控制致動器，並建立簡單的 反饋迴路（feedback loop）。

  • 教學著重於 Webots 功能的正確使用，不探討進階的機器人演算法。

重點內容整理

  • 新世界檔案與控制器建立：

    • 將前章的世界另存為 collision_avoidance.wbt。

    • 透過 File → New → New Robot Controller… 建立新控制器（例如 epuck_avoid_collision）。

    • 在 e-puck 節點中修改 controller 欄位以連結新控制器。

  • e-puck 模型介紹：

    • 配備 8 個紅外距離感測器（ps0 到 ps7），數值範圍 0–4096。

    • 感測器越接近障礙物，輸出值越大。

    • 由 DistanceSensor 節點建模，可透過 webots/distance_sensor.h API 存取。

  • 控制器 API 概念：

    • 提供介面讓使用者能夠存取模擬中的感測器與執行器。

    • 每個設備透過名稱（WbDeviceTag）呼叫 wb_robot_get_device() 取得。

    • 感測器需透過 .enable(time_step) 啟用。

  • 程式語言可用 C、C++、Java、Python，範例使用 Python。

  • 控制邏輯：

    • 機器人向前移動，若前方感測器偵測到障礙物，則轉向無障礙物的一側。

    • 以 UML 狀態機示意控制流程（感測 → 判斷 → 動作）。

機器人設備由 WbDeviceTag 引用。WbDeviceTag 由 wb_robot_get_device 函數檢索。然後，它在每個涉及此設備的函數調用中用作第一個參數。像 DistanceSensor 這樣的傳感器在使用前必須啟用。enable 函數的第二個參數定義了傳感器的刷新率。

結論

  • 控制器的主函數是程式進入點（Python 則從檔案開頭執行）。

  • 在呼叫 wb_robot_init() 之前不可使用任何 Webots API 函數，結束時需呼叫 wb_robot_cleanup()。

  • 各設備以其節點名稱識別，並以 wb_robot_get_device() 取得。

  • 每個控制器程式作為獨立進程執行，與 Webots 主程序之間透過 libController 動態庫通信。
